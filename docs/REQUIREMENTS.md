# Requirements

## Must Have

- Google login through Supabase Auth.
- User profile creation.
- User data isolated by Supabase RLS.
- Attempts, practice submissions, wrong answers, bookmarks, highlights, annotations, todos, learning sessions, and notes stored in DB.
- Approved content only for general learners.
- Admin review workflow for generated or imported content.
- Concept highlights individually removable by clicking highlighted spans; one clear-all button only.
- Personal notes with block-oriented data model.
- SQL practice workspace that labels Oracle plans as simulated/reconstructed unless an actual Oracle sandbox is connected.
- Tests, typecheck, and production build must pass before push/deploy.

## First Quality Content

- 10 questions per subject.
- 3 SQLP practice scenarios.
- At least 3 full concept documents.
- Every item has source type, status, version, answer, explanation, and hints.

## Deferred Or Limited

- Actual isolated Oracle execution sandbox.
- Paid AI usage controls beyond server-only API boundary.
- Full 100/100/100 + 20 expansion until quality gates can verify all generated content reliably.
