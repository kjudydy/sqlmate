## 추출 및 렌더링 주의사항

- 1p, 12p, 20p, 40p, 71p, 93p, 106p, 107p는 텍스트 추출량이 적거나 이미지/도표 중심으로 보였다.
- 일부 페이지는 글자 간격과 인코딩 때문에 추출 텍스트가 불안정했다.
- 도표, ERD, 실행계획, 실기 지문은 텍스트만으로 판단하지 않고 렌더링 이미지를 함께 확인해야 한다.
- PDF 원문, 선택지, 해설, 표를 SQLMate에 그대로 복제하지 않는다.

## 문제은행에 반영할 품질 기준

- 1·2·3과목 필기 문제는 클릭형 객관식으로 유지한다.
- 문제는 문제 본문만 두지 말고 필요할 때 SQL, 표, ERD, 실행 결과, 실행계획, Trace를 함께 제공한다.
- 오답 선택지는 터무니없는 답이 아니라 실제 수험생이 헷갈리는 개념 차이로 구성한다.
- 선택지별 해설은 왜 정답인지와 왜 오답인지 모두 설명한다.
- 동일한 판단 과정을 숫자, 테이블명, 컬럼명만 바꿔 반복하지 않는다.
- SQL 작성, 힌트 작성, 실행계획 유도, 인덱스 설계는 필기 문제가 아니라 SQL Practice로 분리한다.

## 과목별 보강 방향

### 1과목

- 데이터 모델링, 식별자, 관계, 정규화, 반정규화, 성능 데이터 모델링을 표/ERD 판단형으로 보강한다.
- 단순 정의 암기보다 “이 모델에서 어떤 반정규화가 적절한가”, “어떤 식별자 선택이 문제인가”처럼 상황 판단형으로 출제한다.

### 2과목

- SQL 문법 암기보다 실행 결과 추론, GROUP BY/HAVING, JOIN, 서브쿼리, 윈도우 함수의 처리 결과 판단을 강화한다.
- 보기마다 비슷한 SQL을 제시하고 결과 차이를 판단하게 하는 유형을 포함한다.

### 3과목

- 옵티마이저, 통계정보, 인덱스, 조인, 실행계획, SQL Trace, Lock/동시성을 고난도 선택형으로 구성한다.
- Access Predicate와 Filter Predicate, Selectivity와 Cardinality, NL Join과 Hash Join처럼 헷갈리는 개념쌍을 오답으로 적극 활용한다.

## SQL Practice 보강 방향

- 실습 문제는 단순 SQL 입력이 아니라 업무 요구사항, 데이터 분포, 인덱스 구성, 현재 SQL, 실행계획, Trace 요약을 함께 제공한다.
- 모범 SQL만 비교하지 않고 개선 근거, 인덱스 설계, 예상 실행계획, 대안 SQL을 함께 제시한다.
- 실제 Oracle에서 실행하지 않은 실행계획과 Trace는 “설명용 예시”로 표시한다.
- SQL Practice 문제는 PDF의 실기형 사고 과정을 참고하되, 지문과 답안은 SQLMate 자체 문제로 새로 작성한다.

## 현재 SQLMate와의 차이

- SQLMate는 이미 과목별 100문제와 SQL Practice 20문제를 갖고 있으나, 앞으로 신규 배치에서는 PDF처럼 자료 해석형 문제 비중을 더 높여야 한다.
- 기존 배치 생성 가드레일은 객관식 유지와 중복 방지 중심이었고, 이번 PDF 분석 후 ERD/SQL/실행계획/Trace 자료 활용 기준을 추가했다.
- 오답노트는 최근 작업에서 원문 문제, 선택지, 사용자 답, 정답, 선택지별 해설을 복습할 수 있도록 강화되어 PDF식 해설 학습 흐름과 맞아 있다.

## 저작권 및 출처 원칙

- 이 PDF는 출제 스타일과 난이도 분석용 기준 자료로만 사용한다.
- 문제 본문, 선택지, 해설, 표, SQL을 그대로 복사하지 않는다.
- 신규 문제는 평가 개념, 추론 과정, 업무 상황, SQL 구조를 새로 설계한다.
- 콘텐츠에는 사용자 제공 자료 기반, 자체 제작, 검수 필요 등의 출처 유형을 기록할 수 있어야 한다.
